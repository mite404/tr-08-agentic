Knob Component — Bug Demos

Two real bugs from the same component, made interactive

01

The Drag Deadzone

Checking the old angle before computing the new one

Before — Bug

Checks current angle against bounds
then computes new angle — too late

stored angle —
valid range 10° — 256°
excess ← deadzone —
drag back to recover
After — Fixed

Computes new angle first
then clamps — hard stop, no deadzone

stored angle —
valid range 10° — 256°
excess 0° — always

Try This

1On either knob, drag all the way to the top (max).
2On the Before knob, keep dragging fast past the top. The stored angle goes red — you've overshot.
3Now drag back down. Notice the knob doesn't move until you've eaten through the deadzone.
4On the After knob: drag past max — the arc flashes green and the pip stops dead. Reverse instantly.
02

The Volume Persistence Bug

Serializing effective volume instead of raw volume

Before — Bug

Serializes effective volume
Mute writes −∞ into stored value

raw (dragged to) —
stored ← bug here —
played —
⚠ Stored value is −∞ — original lost
After — Fixed

Serializes raw volume only
Mute is playback-only, never stored

raw (dragged to) —
stored ← safe —
played —

Try This

1Drag either knob to a mid or high value.
2Click Mute on the Before side. Watch stored flip to −∞.
3Click Load Beat — simulates deserializing the saved beat. The knob snaps to minimum.
4The original volume is gone. Un-muting can't recover it.
5Repeat on the After side. Load Beat always restores the exact position.
